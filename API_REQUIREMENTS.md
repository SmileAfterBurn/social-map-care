# Документ вимог до API та ШІ-архітектури (v1.3)

## 1. Функціональні вимоги (Functional Requirements)

### 1.1 Персоналізація ШІ (Ms. Thought / Пані Думка)
- **Тон:** Емпатичний, теплий, материнський, з використанням українських культурних маркерів ("серденько", "сонечко").
- **Мова:** Основна українська (літературна та розмовна).
- **Розпізнавання намірів:** Точна класифікація потреб у соціальній допомозі, юридичних запитах та запитах на психологічну підтримку.

### 1.2 Заземлення пошуку (Search Grounding)
- **Верифікація в реальному часі:** Використання `googleSearch` для перевірки поточної доступності гуманітарної допомоги, контактів хабів та годин роботи.
- **Прозорість:** Вилучення та відображення всіх URL-адрес джерел з `groundingChunks` в інтерфейсі користувача.
- **Картографія:** Використання інструменту `googleMaps` для запитів про місцезнаходження, побудову маршрутів та перевірку місць.

### 1.3 Мультимодальність
- **Голосова взаємодія:** Голосовий зв'язок у реальному часі з низькою затримкою за допомогою Gemini Live API.
- **Синтез мовлення (TTS):** Високоякісний природний український голос для всіх відповідей моделі.
- **Транскрипція:** Транскрипція аудіо користувача та моделі в реальному часі для забезпечення інклюзивності.

## 2. Нефункціональні вимоги (Non-Functional Requirements)

### 2.1 Продуктивність (Performance)
- **Затримка (Latency):** Текстові відповіді < 2с; Початкова затримка голосових відповідей < 500мс.
- **Масштабованість:** Здатність обробляти тисячі записів організацій у контексті.
- **Надійність:** Надійні механізми обробки помилок та відкату (наприклад, перехід на текст, якщо аудіо не вдається).

### 2.2 Безпека та конфіденційність (Security & Privacy)
- **Управління ключами API:** Політика нульового розголошення; ключі обробляються виключно через змінні середовища.
- **Захист PII:** Автоматизована генерація кодів справ для маскування особистих даних клієнтів під час перенаправлень.
- **Відповідність:** Дотримання стандартів на кшталт GDPR для обробки гуманітарних даних.

### 2.3 Доступність (Accessibility)
- **Відповідність WCAG:** Контрастний інтерфейс, підтримка екранних читців та можливість навігації клавіатурою.
- **Voice-First дизайн:** Можливість навігації по всьому процесу пошуку допомоги за допомогою голосових команд.
